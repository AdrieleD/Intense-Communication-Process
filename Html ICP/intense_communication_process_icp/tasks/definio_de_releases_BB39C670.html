Task: Definição de Releases
Nesta atividade deverá ser feita a divisão das releases do projeto.
Disciplines: Projeto
Purpose
Fragmentar as tarefas a serem implementadas para o sistema de forma satisfatória.
Relationships
RolesPrimary Performer:
  • Gerente de Projeto
Additional Performers:
  • Programador Junior
Main Description

Nesta atividade o Gerente de Projeto deverá realizar o particionamento do sistema em pequenas releases a serem implementadas. É importante ressaltar que tais releases devem ser definidas de forma a serem viáveis de implementação em uma semana, uma vez que este é tempo sugerido para cada "fase" do jogo.

Um aspecto importante é a definição de prioridade entre as tarefas a serem desenvolvidas, uma vez que as funcionalidades mais importantes devem ser desenvolvidas primeiramente.
