Task: Levantamento e Análise de Requisitos
Nesta atividade é realizado todo o recolhimento de infromações e interpretação do sistema como um todo.
Disciplines: Requisitos
Purpose
Desenvolver um entendimento geral do sistema e suas aplicações e relacionamentos com os usuários.
Relationships
RolesPrimary Performer:
  • Analista de Sistemas
Additional Performers:
  • Gerente de Projeto
Outputs
  • Diagrama de Classes
  • Modelo de Casos de Uso
  • Software Requirement Specification (SRS)
    Main Description

    Durante a Atividade de Requisitos o Analista de Sistemas deve coletar os requisitos junto ao cliente e realizar toda a análise necessária bem como construir os produtos de trabalho esperados. É importante que este trabalho seja realizado de forma mais que satisfatória, uma vez que os resultados de trabalho desta atividade impactarão diretamente no restante do projeto, inclusive na definição de prazos.

    Steps
    Levantamento de Informações e Requisitos
    O Analista deve recolher toda a especificação do projeto.
    Levantamento de Requisitos
    O Analista deve realizar a estruturação dos requisitos do sistema, construindo a SRS da melhor forma possível. Deve-se também estar atento à indicação de requisitos desnecessários e/ou impossíveis de se realizar de acordo com os parâmetros do sistema.
    Análise de Requisitos

    O Analista deve construir os modelos e diagramas esperados de acordo com os requisitos especificados por ele.
