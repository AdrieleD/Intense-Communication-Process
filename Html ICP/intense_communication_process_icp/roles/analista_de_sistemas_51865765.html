Role: Analista de Sistemas
O Analista de sistema é o responsável por coletar os requisitos do sistema e, a partir destes, entender o projeto como um todo da melhor maneira possível.
Role Sets: Papéis
Relationships
Primary Performs
  • Levantamento e Análise de Requisitos
  • Refinamento de Diagramas
Additionally Performs
  • Construção da Wiki
  • Definição de tecnologias
Modifies
  • Diagrama de Classes
  • Modelo de Casos de Uso
  • Software Requirement Specification (SRS)
    Main Description

    O Analista de Sistemas é responsável principalmente pelas seguintes tarefas:

    • Coletar, identificar e analisar os requisitos junto ao cliente.
    • Produzir diagramas e refiná-los posteriormente. Para a produção de tais diagramas, sugere-se o Modelo de Casos de Uso e o Diagrama de Classes.

    É importante lembrar que o Modelo de Casos de Uso deve ser refinado a partir do Diagrama de Classes. Por sua vez, deve ser produzido ainda na Atividade de Requisitos, um Diagrama de Classes de Domínio, que deverá ser refinado posteriormente.

    Staffing
    Skills

    Um Analista de Sistemas deve preferencialmente possuir as seguintes habilidades:

    • Rápido entendimento de um sistema a partir de requisitos, muitas vezes confusos e incompletos.
    • Facilidade em abstrair informações importantes.
    • Ser uma pessoa comunicativa.
