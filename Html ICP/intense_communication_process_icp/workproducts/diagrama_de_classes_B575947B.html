Description
Main Description

O Diagrama de Classes é composto de duas fases, a primeira delas devendo ser realizada logo na Atividade de Requisitos. Esta primeira fase é conhecida como Diagrama de Classes de Domínio, uma vez que somente as classes, seus relacionamentos e relativas multiplicidades são analisadas. O Diagrama de Classes de Domínio leva em consideração Regras de Negócio e não deve possuir nenhuma característica relativa à implementação, um,a vez que esta ainda será propriamente analisada na Atividade de Projeto.

Uma vez que as características técnicas de desenvolvimento tenham sido bem analisadas na Atividade de Projeto, deve-se realizar um refinamento do Diagrama de Classes de Domínio, incluindo todos os tipos e características próprias da linguagem de programação, por exemplo.

Impact of not havingOs desenvolvedores podem se confundir com o relacionamento entre as classes, podem julgar outras classes necessárias quando estas não o são, entre outros problemas de interpretação e design do sistema. Estes problemas podem levar à uma maior complexidade do sistema como um todo, podendo também aumentar o custo do projeto.
