Artifact: Software Requirement Specification (SRS)
Este artefato é gerado na disciplina de Requisitos. Trata-se do produto de interações entre o cliente e o gerente de projeto, que é o principal responsável pela produção de tal artefato (podendo ser auxiliado pelo analista de sistema).

Description
Main Description A SRS consiste em uma lista de requisitos funcionais e não funcionais que descrevem como o sistema deve funcionar baseados em aspectos técnicos e não técnicos. É válido ressaltar que este artefato pode, e deve, ser refinado, à medida que as iterações ocorrem.
